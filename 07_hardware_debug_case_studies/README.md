# Hardware Debug Case Studies

This section documents representative hardware debugging scenarios
demonstrating structured fault isolation, measurement-driven analysis,
and reliability-focused engineering methodology.

## Included Cases

1. **Post-Silicon MEMS Timing Instability**
   - Thermal corner validation
   - Supply noise correlation
   - PLL instability root cause

2. **PCB Bring-Up – 3.3V Rail Collapse**
   - Regulator oscillation
   - Load isolation methodology
   - Solder bridge identification

3. **Board Won’t Power On – VIN Short**
   - Resistance-based fault detection
   - Thermal localization
   - MLCC fracture root cause

All examples are NDA-safe and structured to reflect real engineering workflows.
